Comparaison : Subnetting Fixe vs VLSM

Cas Tokyo / New York (Page 11) :

Le but est de diviser le réseau 172.17.0.0 /16 en fonction des besoins réels pour éviter le gaspillage.

- Administration : 100 Postes -> Besoin : /25 (128 IP)
- Production : 2000 Postes -> Besoin : /21 (2048 IP)
- DMZ : 20 Serveurs -> Besoin : /27 (32 IP)
- Liaisons WAN : 2 IP -> Besoin : /30 (4 IP)

Le Cas n°5 (Page 9) : Entreprise à 3 départements

Réseau de départ : 193.250.17.0 /24

  • Admin (25 hôtes) : On utilise un /27 (Bloc de 32 IP).
  • Commercial (15 hôtes) : On utilise un /27 (Bloc de 32 IP).
  • Production (20 hôtes) : On utilise un /27 (Bloc de 32 IP).

Justification : Le masque /27 est le plus petit masque capable de contenir 25 hôtes (2^5 - 2 = 30 adresses utilisables).
