1. Les Bases de l'IPv4

Une adresse IPv4 fait 32 bits (4 octets). Elle est divisée en Partie Réseau (NetID) et Partie Hôte (HostID).

Exemple : 192.168.1.10 /24
Masque : 255.255.255.0
Réseau : 192.168.1.0

Les Classes (Rappel PDF p.3)

  • Classe A : 1 à 126 (/8)
  • Classe B : 128 à 191 (/16)
  • Classe C : 192 à 223 (/24)
  • Loopback : 127.0.0.1 (interdit)

2. Le Subnetting (Calcul)

Nombre de Machines

Calculer combien d'ordinateurs peuvent tenir dans un sous-réseau.

2n - 2

n = nombre de bits à 0 dans le masque.

Le Nombre Magique

Pour trouver le pas entre chaque sous-réseau rapidement.

256 - [Octet Masque]

Ex: Pour /26 (192), 256-192 = 64.

Broadcast

L'adresse de diffusion est toujours la dernière adresse du bloc.

Next Réseau - 1

Tous les bits hôtes sont à 1.

3. La Stratégie VLSM (p.13 du PDF)

Le VLSM permet de découper un réseau en tailles variables.

RÈGLE D'OR : On commence toujours l'attribution par le réseau qui a besoin du plus grand nombre d'hôtes, puis on descend vers le plus petit (Liaisons WAN /30 en dernier).

Exemple de hiérarchie :

  1. 1 LAN Production (120 hôtes) - /25
  2. 2 LAN Bureaux (60 hôtes) - /26
  3. 3 LAN WiFi (25 hôtes) - /27
  4. 4 Liaisons WAN (2 hôtes) - /30
