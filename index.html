1. Le concept du VLSM

La création de base de sous-réseaux est suffisante pour les petits réseaux, mais le VLSM (Variable Length Subnet Masking) est indispensable pour les grands réseaux d'entreprise.

Principe Clé

Le VLSM consiste à créer des sous-réseaux d'un sous-réseau. On optimise ainsi chaque adresse IP.

Compatibilité

Les protocoles "Classfull" comme RIPv1 ne le supportent pas. Il faut utiliser RIPv2, EIGRP ou OSPF.

Avantages du VLSM :

  • Utilisation efficace de l'espace (moins de gaspillage).
  • Plusieurs longueurs de masques sur un même réseau.
  • Supporte le résumé de routes (agrégation).

3. Classes & Attribution FAI

Classes standards

  • 10.0.3.45 Classe A / Privée
  • 172.16.0.1 Classe B / Privée
  • 127.0.0.1 Réservée (Loopback)
  • 224.0.0.1 Multicast

Attribuable par un FAI ?

Un FAI ne peut attribuer que des adresses publiques.

  • 10.0.1.2 (Privée)
  • 115.3.4.5 (Publique)
  • 192.168.34.73 (Privée)

4. Calcul de Subnetting

Cas : Réseau Classe B avec Masque 255.255.240.0

Q1 : Combien de bits empruntés à la partie hôte ?

240 en binaire = 11110000
Réponse : 4 bits ont été empruntés.

Q2 : Combien de sous-réseaux utilisables ?

Formule : 2n (où n est le nb de bits empruntés)
24 = 16 sous-réseaux.

5. Étude de cas : Tokyo vs New York

Découpage VLSM complexe

Pour le réseau de Tokyo, nous avons différents besoins :

2000 Postes Production Masque /21
100 Postes Admin Masque /25
20 Serveurs DMZ Masque /27

Pourquoi utiliser des masques différents ?

Si on utilisait le même masque pour tout le monde (basé sur les 2000 postes), on gaspillerait des milliers d'adresses IP pour la DMZ de 20 serveurs.

6. Interpréter la Table de Routage

Analyse de la commande show ip route :

LAB-B# show ip route
...
R 204.204.7.0/24 [120/1] via 199.6.13.2, 00:00:09, Serial0
C 219.17.100.0/24 is directly connected, Ethernet0
R 192.5.5.0/24 [120/1] via 201.100.11.1, 00:00:04, Serial1
C 199.6.13.0/24 is directly connected, Serial0
Comprendre les codes :
  • C : Réseau directement connecté.
  • R : Réseau appris via le protocole RIP.
  • [120/1] : Distance administrative / Métrique.
Déduction Topologie :

Le routeur possède au moins deux interfaces série (Serial0, Serial1) et une interface Ethernet (Ethernet0).
